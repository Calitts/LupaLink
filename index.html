Como funciona?

1.1 Validação Inicial da URL
Ao receber uma URL, a API realiza uma tentativa de conexão com o endereço fornecido. Essa etapa inicial tem como finalidade verificar a validade da URL e identificar possíveis erros críticos, como a ausência de certificado SSL.
1.2 Fragmentação da URL
Após a conexão inicial, a URL é fragmentada em seus principais componentes: domínio, parâmetros (query strings), protocolo, entre outros. Uma função específica é responsável por resolver o domínio até o endereço IP correspondente da aplicação.
1.3 Extração de Características do Site
Com os dados iniciais processados, a API inicia a coleta de diversas características do site analisado. Nesse ponto, o comportamento da API varia de acordo com o valor do parâmetro verif, recebido na query da requisição:
Caso verif = 1:
A API realiza um processo de crawling, no qual o scanner acessa a URL principal e coleta todas as URLs secundárias que contenham parâmetros. Essas URLs são então analisadas em busca de possíveis vulnerabilidades. Os resultados desses testes não são divulgados ao usuário, com o objetivo de evitar o uso mal-intencionado da ferramenta. A única informação retornada é se o site analisado é ou não recomendado para o fornecimento de dados pessoais.


Caso verif = 0:
Toda a etapa de crawling e análise de vulnerabilidades é ignorada, seguindo diretamente para as demais verificações.


1.4 Consulta ao Abuse IPDB
A API realiza, em seguida, uma consulta ao banco de dados Abuse IPDB, de onde extrai informações como:
Quantidade de denúncias (reports) associadas ao IP;


Se o IP é público ou privado;


Se o IP pertence a alguma lista de confiança (whitelist).


Esses dados auxiliam na avaliação do risco associado ao endereço analisado.
1.5 Verificação de Hospedagem e Proteções
A ferramenta também busca identificar onde o site está hospedado e se há algum mecanismo de segurança presente, como firewalls ou serviços de proteção de tráfego.
1.6 Consulta ao Whois
É realizada uma consulta ao serviço Whois, que fornece informações relacionadas ao domínio do website, como:
Nome e visibilidade do proprietário;


Datas de criação e expiração do domínio;


Status do registro.


Esses elementos são considerados no cálculo da confiabilidade do site.
1.7 Avaliação do Site via Trustpilot
Por fim, a API realiza uma busca na plataforma Trustpilot para coletar a nota média atribuída pelos usuários ao site analisado. Essa avaliação pública também contribui para a pontuação final.


Quem somos?

Somos um grupo de estudantes do curso de Ciência da Computação da Universidade Federal do Ceará – Campus Quixadá, envolvidos no projeto de extensão “Autodefesa Digital: convivendo com armadilhas na internet”. Nosso objetivo é proporcionar aos usuários da internet uma ferramenta fácil e intuitiva para verificar links suspeitos, promovendo uma navegação mais segura no ambiente digital.
